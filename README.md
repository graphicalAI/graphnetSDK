# GraphNet SDK

Утилита командной строки (SDK) для быстрого создания, компиляции и тестирования пользовательских расширений (плагинов) для визуального редактора **graphNet**.

## Быстрый старт (Workflow)

С помощью SDK вы можете легко сгенерировать шаблон расширения, написать свою логику, а затем скомпилировать его в готовый `.zip` архив для установки в редактор.

### 1. Инициализация расширения (init)
Создаёт базовую структуру расширения (папку с нужными файлами: `__init__.py` и `extension_config.py`).

```bash
uv run graphnet_sdk.py init ext_MyAwesomeNode
```
*(Вместо `uv run` вы можете использовать стандартный `python`, если у вас установлены все зависимости)*

### 2. Написание логики
Перейдите в созданную папку (например, `ext_MyAwesomeNode`). 
- Откройте `extension_config.py` и опишите логику работы вашего узла (например, обработку данных, применение слоев нейросетей или алгоритмов).
- Для настройки входных и выходных параметров отредактируйте конфигурацию аннотаций в `__init__.py`.

### 3. Компиляция (compile)
Упаковывает ваше расширение в единый `.zip` архив, который можно публиковать или передавать другим пользователям.

```bash
uv run graphnet_sdk.py compile ext_MyAwesomeNode
```
На выходе появится файл `ext_MyAwesomeNode.zip`. Вы также можете явно указать имя выходного файла:
```bash
uv run graphnet_sdk.py compile ext_MyAwesomeNode -o MyPlugin.zip
```

### 4. Установка в проект (run)
Распаковывает и устанавливает скомпилированный архив напрямую в директорию `Extensions/` основного приложения graphNet.

```bash
uv run graphnet_sdk.py run ext_MyAwesomeNode.zip --overwrite
```
*Флаг `--overwrite` позволяет перезаписать расширение, если вы устанавливаете его повторно во время разработки.*

**Всё готово!** Запустите основное приложение graphNet, и ваш новый узел автоматически появится в меню графического редактора.
